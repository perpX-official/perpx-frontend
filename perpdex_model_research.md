# Perpetual DEX Model Research

## AMM-Based Model

### 概要
AMM（Automated Market Maker）型Perpetual DEXは、仮想AMM（vAMM）を使用して価格発見を行うモデル。実際のスポット資産の交換ではなく、名目上の先物契約がAMMで取引されたと仮定し、決済時に残高のみを精算する。

### 主な特徴

**利点**:
1. 誰でも簡単にマーケットメイカーとして参加可能（x*y=K方程式で自動価格決定）
2. 新しい取引ペア追加のコストが最も低い
3. 完全にオンチェーンで価格発見が可能（理論上）
4. DeFiプロトコルとの高い組み合わせ可能性（Composability）

**課題**:
1. **Mark Price vs Index Price**: 現在は取引量が少ないため、AMMの価格（Mark Price）が市場価格（Index Price）に追従する必要がある
2. **Funding Cost**: 価格追従のためのアービトラージャーへのインセンティブコストが常に発生
3. **仮想K値の設定**: 適切な流動性深度（K値）の設定が困難
   - K値が大きすぎる：価格が動かず、資金調達コストが増大
   - K値が小さすぎる：スリッページが高く、ユーザーが離脱

### アーキテクチャ

**AMM自体がHouse（ディーラー）として機能**:
- すべてのトレーダーとアービトラージャーのポジションのカウンターパーティとなる
- オープンインタレストの不均衡による資金調達コスト/利益はプロトコルに帰属
- オーダーブック型（ブローカーモデル）とは根本的に異なる

**現在の価格フロー**:
```
CEX先物 → CEXスポット → Oracle → AMM Perp DEX Index Price <> Mark Price
```

### 代表的なプロジェクト

1. **Perpetual Protocol**
2. **Drift Protocol**
3. **GMX**（ハイブリッド要素あり）

### 実装の複雑さ

- スマートコントラクト: ★★★★☆（複雑）
- 流動性要件: ★★★★★（非常に高い）
- 初期開発コスト: $200K〜$500K

## Order Book-Based Model

### 概要
Order Book型Perpetual DEXは、従来の中央集権型取引所（CEX）と同様の仕組みを分散型フレームワークで実装したモデル。買い注文と売り注文が価格と時間優先順位に基づいてマッチングされる。

### 主な特徴

**利点**:
1. **豊富な流動性**: プロフェッショナルなマーケットメイカーによる流動性提供
2. **低スリッページ**: 正確な注文マッチングにより、スリッページが低い
3. **参入障壁が低い**: CEXに似ているため、すべてのクラスのトレーダーが採用・操作しやすい
4. **価格発見**: 最高買い注文と最低売り注文によって価格が決定される

**課題**:
1. **完全オンチェーンの困難性**: スループット制限、ブロックチェーンレイテンシ、フロントランニング、操作脆弱性
2. **マーケットメイカー依存**: 新しい資産追加時にプロフェッショナルなマーケットメイカーの存在が必要
3. **オフチェーン要素**: 現状、完全な分散化のためにはオフチェーンオーダーブックが必要

### アーキテクチャ

**ブローカーモデル**:
- トレーダー同士が反対のポジションを取る
- 取引所は独自のマーケットメイキング以外は仲介役のみ
- AMM型のようにHouseとして機能しない

### 代表的なプロジェクト

1. **Hyperliquid**
2. **Zeta**
3. **LogX**
4. **dYdX**（ハイブリッド要素あり）

### 実装の複雑さ

- スマートコントラクト: ★★★★★（非常に複雑）
- 流動性要件: ★★★★★（マーケットメイカー必須）
- 初期開発コスト: $300K〜$800K

## Hybrid Model（オフチェーンオーダーブック + オンチェーン決済）

### 概要
取引マッチングはオフチェーンで行い、トランザクション実行とそれ以外のプロセスはオンチェーンで実行するハイブリッドアプローチ。

### 主な特徴

**利点**:
1. **DeFiのメリット**: カストディ、透明性、セキュリティを確保
2. **CEXレベルの速度**: オフチェーンマッチングによる高速化
3. **MEVリスク軽減**: オフチェーンマッチングによりMEV攻撃を軽減
4. **低スリッページ**: 高頻度マーケットメイカーによる深い流動性プール
5. **効率的な価格発見**: CEXと同様の迅速な価格更新と注文マッチング

**アーキテクチャ**:
- オフチェーン: オーダーブック、マッチングエンジン
- オンチェーン: 決済、カストディ、最終的なトランザクション実行

### 代表的なプロジェクト

1. **dYdX**（v3以降）
2. **Aevo**
3. **ParaDEX**

### 実装の複雑さ

- スマートコントラクト: ★★★★☆（複雑）
- オフチェーンインフラ: ★★★★★（非常に複雑）
- 流動性要件: ★★★★★（マーケットメイカー必須）
- 初期開発コスト: $400K〜$1M+

## Oracle-Based Model（補足）

### 概要
外部オラクルから市場価格を取得し、独自の価格発見メカニズムを持たないモデル。

### 主な特徴

**利点**:
1. **シンプルな実装**: 価格発見メカニズムが不要
2. **低い流動性要件**: 外部価格に依存するため、独自の流動性が不要
3. **迅速なローンチ**: 開発期間が短い

**課題**:
1. **オラクル依存**: 外部データソースへの依存
2. **価格操作リスク**: オラクル攻撃の可能性
3. **限定的な資産**: 十分な取引量と市場深度がある資産のみ

### 代表的なプロジェクト

1. **GMX**（GLP流動性プール + Chainlinkオラクル）
2. **Gains Network**
