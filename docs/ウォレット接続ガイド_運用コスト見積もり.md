## 1. ウォレット接続の仕組み

### 全体アーキテクチャ

ウォレット接続は、複数のレイヤーで構成されています。

```
┌─────────────────────────────────────────────────────────────────┐
│                      WalletConnect Inc.                          │
│              (プロトコル・インフラを提供する会社)                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌────────────────┐   ┌────────────────┐   ┌────────────────┐   │
│  │  WalletConnect │   │    AppKit      │   │   その他SDK    │   │
│  │   Protocol     │   │ (旧Web3Modal)  │   │                │   │
│  │   (通信規格)    │   │   (公式UI)     │   │                │   │
│  └────────────────┘   └────────────────┘   └────────────────┘   │
│          ↑                    ↑                    ↑            │
└──────────┼────────────────────┼────────────────────┼────────────┘
           │                    │                    │
           │     ┌──────────────┴──────────────┐     │
           │     │      サードパーティUI        │     │
           │     ├─────────────┬───────────────┤     │
           │     │ RainbowKit  │  ConnectKit   │     │
           │     │ (Rainbow社) │  (Family社)   │     │
           │     └─────────────┴───────────────┘     │
           │                 ↓                       │
           └─────────────────┴───────────────────────┘
                             │
                        あなたのDApp
                         (PerpX)
```

### 通信フロー

ユーザーがウォレットを接続する際の流れは以下の通りです。

| ステップ | 処理内容 |
|---------|---------|
| 1 | ユーザーが「Connect Wallet」ボタンをクリック |
| 2 | RainbowKit/AppKitがウォレット選択モーダルを表示 |
| 3 | ユーザーがウォレット（MetaMask等）を選択 |
| 4 | WalletConnect Protocolを通じて接続リクエストを送信 |
| 5 | ウォレットアプリが接続承認を要求 |
| 6 | ユーザーが承認 → 接続完了 |
| 7 | wagmiがウォレットアドレス・残高等を取得 |

## 2. 用語解説

### SDK（Software Development Kit）

**定義:** 複雑な機能を簡単に使えるようにパッケージ化した「開発キット」

**料理に例えると:**
- SDK = カレールー
- ルーを使えば、スパイスの配合を知らなくてもカレーが作れる
- SDKを使えば、ウォレット接続の複雑な処理を知らなくても実装できる

### wagmi

**定義:** Ethereumとやり取りするための**Reactフック集**

```typescript
// wagmiを使った例
const { address, isConnected } = useAccount();  // アドレス取得
const { connect } = useConnect();                // 接続処理
const { data: balance } = useBalance();          // 残高取得
```

**料理に例えると:**
- wagmi = 基本の調味料セット（醤油、みりん、酒）
- これだけでも料理はできるが、毎回自分で味付けを考える必要がある

### viem

**定義:** Ethereumブロックチェーンとの低レベル通信ライブラリ

wagmiの内部で使用されており、直接触ることは少ないですが、以下のような処理を担当します。

- トランザクションの送信
- コントラクトの呼び出し
- ブロックデータの取得
- 署名の検証

### サードパーティ

**定義:** 「第三者」の意味。WalletConnect社以外の会社が作ったもの。

| 種類 | 提供元 | 製品例 |
|------|-------|--------|
| ファーストパーティ | WalletConnect Inc. | AppKit (Web3Modal) |
| サードパーティ | Rainbow社 | RainbowKit |
| サードパーティ | Family社 | ConnectKit |

### WalletConnect Protocol

**定義:** ウォレットとDApp間の通信規格

QRコードやディープリンクを使って、異なるデバイス間でも安全にウォレット接続を実現する技術です。

**特徴:**
- エンドツーエンド暗号化
- オープンソース
- 業界標準として広く採用

## 3. SDK比較

### RainbowKit vs AppKit vs 独自実装

| 項目 | RainbowKit | AppKit (Web3Modal) | 独自実装 |
|------|------------|-------------------|---------|
| **提供元** | Rainbow社 | WalletConnect Inc. | 自社 |
| **UI表示** | 「RainbowKit」ブランド | 「WalletConnect」ブランド | 完全カスタム |
| **開発時間** | 数時間 | 数時間 | 2〜4週間 |
| **カスタマイズ性** | 高い | 中程度 | 完全自由 |
| **対応チェーン** | EVM系のみ | EVM + Solana + Bitcoin | 自由 |
| **メンテナンス** | ライブラリ更新のみ | ライブラリ更新のみ | 全て自社 |
| **新ウォレット対応** | 自動 | 自動 | 手動実装 |

### 主要DEXの採用状況

| DEX | 使用SDK | 備考 |
|-----|--------|------|
| **Uniswap** | 独自実装 | 以前はRainbowKit使用 |
| **SushiSwap** | RainbowKit | - |
| **PancakeSwap** | 独自実装 | - |
| **dYdX** | 独自実装 | - |
| **GMX** | RainbowKit | 大手Perp DEX |
| **Hyperliquid** | 独自実装 | - |
| **Vertex** | AppKit | - |

### なぜ大手DEXは独自実装するのか

| 理由 | 詳細 |
|------|------|
| **ブランディング** | 「RainbowKit」「WalletConnect」のロゴを出したくない |
| **完全なUI制御** | 自社のデザインシステムに完全に合わせたい |
| **機能の最適化** | 不要な機能を削除、必要な機能だけに絞りたい |
| **パフォーマンス** | 余計なコードを読み込まない |
| **開発リソース** | 専任のフロントエンドチームがいる |

### なぜ中小DEXはRainbowKit/AppKitを使うのか

| 理由 | 詳細 |
|------|------|
| **開発速度** | 数時間で実装完了 |
| **バグが少ない** | 多くのプロジェクトで検証済み |
| **メンテナンス不要** | ライブラリ更新だけでOK |
| **新ウォレット対応** | 自動で新しいウォレットに対応 |
| **コスト** | 開発者1人でも実装可能 |

## 4. RainbowKit継続時の設定

### 現在のPerpX構成

| パッケージ | バージョン | 役割 |
|-----------|-----------|------|
| `@rainbow-me/rainbowkit` | ^2.2.9 | ウォレット接続UI |
| `wagmi` | ^2.18.2 | Reactフック集 |
| `viem` | ^2.38.4 | 低レベル通信 |

### 設定ファイル

**ファイルパス:** `client/src/config/wagmi.ts`

```typescript
import { getDefaultConfig } from '@rainbow-me/rainbowkit';
import { mainnet, arbitrum, base, optimism } from 'wagmi/chains';

const WALLETCONNECT_PROJECT_ID = 'あなたのProject ID';

export const config = getDefaultConfig({
  appName: 'PerpX',
  projectId: WALLETCONNECT_PROJECT_ID,
  chains: [mainnet, arbitrum, base, optimism],
  ssr: false,
});
```

### カスタマイズ可能な項目

| 項目 | 設定方法 |
|------|---------|
| 対応チェーン | `chains`配列を変更 |
| アプリ名 | `appName`を変更 |
| テーマカラー | `RainbowKitProvider`の`theme`プロパティ |
| 表示ウォレット | `wallets`配列で指定 |

## 5. WalletConnect Project ID について

### Project IDとは

WalletConnect Cloudで発行される、あなたのDAppを識別するための一意のIDです。全てのWalletConnect対応SDK（RainbowKit、AppKit、ConnectKit等）で共通して使用します。

### RainbowKitでもWalletConnect公式のProject IDは使えるか

**はい、使えます。**

Project IDはWalletConnect社が発行するもので、どのSDKを使っても同じIDを使用します。

```
WalletConnect Cloud (Project ID発行)
        ↓
   ┌────┴────┐
   ↓         ↓
RainbowKit  AppKit  ← どちらも同じProject IDを使用
```

### Project ID取得手順

1. [WalletConnect Cloud](https://cloud.walletconnect.com/) にアクセス
2. アカウント作成（GitHub/Google連携可能）
3. 「New Project」をクリック
4. プロジェクト名を入力（例：PerpX）
5. Project IDが発行される
6. `client/src/config/wagmi.ts`のIDを差し替え

### 料金体系

| プラン | 月額 | リクエスト数 |
|-------|------|-------------|
| Free | $0 | 100,000/月 |
| Growth | $99 | 1,000,000/月 |
| Scale | $499 | 10,000,000/月 |
| Enterprise | 要問合せ | 無制限 |

**注意:** 無料プランでも月10万リクエストまで対応可能です。一般的なDEXであれば、ローンチ初期は無料プランで十分です。

## 6. 運用コスト見積もり

### 前提条件

- フロントエンド + 管理画面のみ（DEXバックエンド/スマートコントラクトは含まない）
- タスク報酬システムの運用
- ユーザー数に応じた3パターンで見積もり

### コスト内訳

#### 1. ホスティング費用

| サービス | 用途 | 少人数 (〜1,000人) | 中規模 (〜10,000人) | 大規模 (〜100,000人) |
|---------|------|-------------------|--------------------|--------------------|
| **フロントエンド** | Vercel / Cloudflare Pages | $0 | $20/月 | $50〜100/月 |
| **管理画面サーバー** | Railway / Render / VPS | $5〜10/月 | $20〜50/月 | $100〜200/月 |
| **データベース** | PlanetScale / TiDB | $0 (Free Tier) | $29/月 | $99〜199/月 |

#### 2. 外部サービス費用

| サービス | 用途 | 少人数 | 中規模 | 大規模 |
|---------|------|--------|--------|--------|
| **WalletConnect** | ウォレット接続 | $0 | $0 | $99/月 |
| **ドメイン** | 独自ドメイン | $10〜15/年 | $10〜15/年 | $10〜15/年 |
| **SSL証明書** | HTTPS | $0 (自動) | $0 (自動) | $0 (自動) |

#### 3. オプション費用

| サービス | 用途 | 費用 |
|---------|------|------|
| **X API** | ツイート自動検証 | $100/月 (Basic) |
| **Discord Bot** | サーバー管理 | $0〜5/月 |
| **CDN** | 画像配信高速化 | $0〜20/月 |
| **監視ツール** | エラー監視 | $0〜29/月 |

### 月額コスト早見表

| 規模 | ユーザー数 | 月額コスト | 年額コスト |

1. **WalletConnect Cloud**でProject IDを取得
